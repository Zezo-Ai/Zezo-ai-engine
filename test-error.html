Ways to Test Chatbot Errors

1. Invalid Bot ID

Use this shortcode with a non-existent bot ID:

[mwai_chatbot id="99999"]

2. Temporary Code Modification

Add this line in ChatbotContext.js after line 847 to force an error:

// Temporary: Force an error for testing
if (textQuery.toLowerCase().includes('error')) {
    throw new Error('Test error: This is a simulated error for testing the error display');
}

3. Invalid API Key

Go to AI Engine > Settings and temporarily change your OpenAI API key to something invalid like "sk-invalid"

4. Network Error

Use browser DevTools:

  1. Open DevTools (F12)
  2. Go to Network tab
  3. Set throttling to "Offline"
  4. Try to send a message

5. Block the Request

In DevTools Network tab:

  1. Right-click on the domain
  2. Select "Block request domain"
  3. Try to send a message

6. Test with Special Messages

You could modify the server-side PHP to return errors for specific messages:

// In your PHP endpoint handler
if (strpos($message, 'test error') !== false) {
    return new WP_Error('test_error', 'This is a test error message');
}
